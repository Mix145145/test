# QFA Storage (Android)

Мобильное Android-приложение для **учёта вещей дома**: где что лежит, как это быстро найти, и как связывать места хранения с QR-кодами.

> Важно: в проекте остались старые названия классов и пакетов (`aoi`, `inspection`, `markup`) из предыдущего направления. Это **наследие**, а не актуальная бизнес-цель.
> Текущая цель продукта — домашний учёт вещей и навигация по местам хранения.

## Что умеет приложение

- **Поиск вещей и мест хранения** по названию, категориям, типам хранения и QR-идентификаторам.
- **Карта хранения** (комнаты/зоны/контейнеры) для визуального представления, где лежат вещи.
- **QR-сценарии**:
  - сканирование QR,
  - генерация QR для предметов,
  - пакетная генерация кодов и экспорт.
- **Локальное хранение данных** на устройстве (Room, DataStore), без обязательного облака.
- Дополнительные экраны (модели/камера/разметка) присутствуют как техдолг и используются частично.

## Технологический стек

- **Kotlin + Jetpack Compose**
- **Hilt** (DI)
- **Room** (локальная БД)
- **DataStore** (настройки)
- **CameraX** (камера)
- **ZXing** (QR)
- ONNX Runtime (наследие от предыдущего направления)

## Минимальные требования

- Android SDK: `minSdk 26`
- Целевая версия: `targetSdk 34`
- JDK 17

## Как запустить

1. Установите Android Studio (Hedgehog+ желательно) и JDK 17.
2. Откройте проект в Android Studio.
3. Дождитесь синхронизации Gradle.
4. Запустите приложение на эмуляторе или физическом устройстве (Android 8+).

CLI-вариант сборки:

```bash
./gradlew assembleDebug
```

## Текущая структура (кратко)

- `app/src/main/java/com/sbm/aoi/ui/storage` — UI домашнего учёта (поиск, карта, QR).
- `app/src/main/java/com/sbm/aoi/storage` — модели/DAO/БД для слоя хранения.
- `app/src/main/java/com/sbm/aoi/data` — репозитории и модели данных.
- `tools/` — утилиты для датасетов/конвертации моделей (наследие AI-части).

## Почему встречаются старые термины

Проект эволюционировал: ранние итерации были связаны с визуальной инспекцией и моделями детекции.
Сейчас продукт переориентирован на бытовой сценарий — **«где дома лежат мои вещи»**.

Если будете продолжать разработку, рекомендуется постепенно:

- переименовать package/классы (`aoi` → более нейтрально),
- отделить legacy AI-модули в отдельный модуль,
- актуализировать экраны навигации под storage-first сценарий.
